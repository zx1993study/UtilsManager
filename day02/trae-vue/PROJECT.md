# 后台管理系统 - 项目说明文档

## 🎯 项目概述

这是一个功能完善的后台管理系统，采用现代化的技术栈构建，提供了完整的系统管理、接口测试、页面管理、流程管理和报告统计等功能模块。

## 📋 技术栈详情

### 核心技术
- **Vue 3**: 采用 Composition API，更好的逻辑组织和代码复用
- **Vite**: 极速的开发服务器启动和热更新
- **Element Plus**: 企业级 UI 组件库
- **Pinia**: Vue 官方状态管理方案
- **Vue Router**: 路由管理，支持权限控制
- **Axios**: HTTP 请求封装
- **ECharts**: 数据可视化图表

### 图标库
- **@element-plus/icons-vue**: Element Plus 图标库

## 🎨 统一设计规范

### 按钮规范
```vue
<!-- 使用 CommonButton 或 Element Plus Button -->
<el-button type="primary">主要按钮</el-button>
<el-button type="success">成功按钮</el-button>
<el-button type="danger">危险按钮</el-button>
```

### 输入框规范
- 统一宽度：200px（搜索）、100%（表单）
- 清晰的占位符：`请输入XXX` / `请选择XXX`
- 支持清空功能

### 弹窗规范
- 统一使用 `CommonDialog` 组件
- 默认宽度：600px
- 底部按钮：取消、确定
- 自动表单验证

### 表格规范
- 统一使用 `CommonTable` 组件
- 边框 + 斑马纹
- 居中对齐
- 序号列
- 选择框（可选）
- 操作列：编辑、复制、删除

### 提示规范
```

ElMessage.success('操作成功')

// 错误提示
ElMessage.error('操作失败')

// 确认对话框
```

## 🔨 扩展开发

### 添加新模块
1. 在 `src/views/` 创建目录和页面
2. 在 `src/router/index.js` 添加路由
3. 菜单会自动显示

### 自定义API

#### API目录结构规范

所有API接口统一在 `src/api/` 目录下管理，采用**按菜单模块分组**的组织方式：

1. **一级分类 - 菜单模块**：每个左侧导航栏的一级菜单对应一个文件夹
2. **二级分类 - 页面API**：每个菜单下的页面对应一个独立的API文件
3. **命名规范**：文件名格式为 `{模块名}-{页面功能}.js`

**示例结构**：
```
src/api/
├── api/                        # 接口管理菜单模块
│   ├── api.js                  # 接口列表页面API
│   ├── api-template.js         # 接口模板页面API
│   ├── api-testcase.js         # 接口测试用例页面API
│   └── api-result.js           # 接口测试结果页面API
├── system/                     # 系统管理菜单模块
│   ├── user.js                 # 用户管理页面API
│   ├── dept.js                 # 部门管理页面API
│   └── role.js                 # 角色管理页面API
└── project/                    # 项目管理菜单模块
    └── project.js              # 项目列表页面API
```

#### API方法命名规范

- `getList`: 获取列表数据
- `getDetail`: 获取详情数据
- `add`: 新增数据
- `update`: 更新数据
- `delete`: 删除数据

#### 使用示例

```
// 导入特定模块的API
import { getApiList, addApi } from '@/api/api/api'
import { getUserList, addUser } from '@/api/system/user'

// 调用API
const res = await getApiList({ pageNum: 1, pageSize: 10 })
```

#### 配置后端地址

修改 `src/utils/request.js`:
```
const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',
  timeout: 10000
})
```

建议在 `.env.development` 和 `.env.production` 中配置不同的API基础路径。

### 添加新组件
在 `src/components/` 创建组件，按需引入使用。
